Gameplay

This game is continuously running rounds forever. New players start with 0 (battle, round) (wins, losses, ties), score, and streak. For convenience, players can reset their state, which acts as if they reconnected.

Resetting is generally disadvantageous, as it ruins streaks.

Round

Rounds have options. The game randomly selects one player, whose option choices are used. If an option is marked as “don't care”, the value is undefined, but the game reuses the value from the previous round or randomly select something.

In a round, these phases occur:

  1. Move selection
  2. Damage calculation
  3. Battle execution
  4. Score calculation
  5. Result evaluation

Move selection

Every player has four options: pick Rock, Paper, Scissors, or nothing. If nothing is selected, the game replaces the player's choice: one of the first three is randomly selected, or if the player has a streak bonus, an optimal choice is made automatically.

This phase lasts 5 seconds (3 seconds if Blitz is on) plus the last phase of a previous round, if applicable.

Once every player has chosen rock, paper, or scissors, the same move applies to each player for the entire round.

Damage calculation

When two moves are the same, the result is always a tie.

By default,

  • Rock beats Scissors
  • Scissors beats Paper
  • Paper beats Rock

If Inverted is on, the defaults are reversed. If Count is on, when a pair of moves is chosen by a different number of players, the one with more players never loses (see below for more details).

Battle execution

Unless modified by Team or Survivor, all N(N-1)/2 pairs of N participating players battles each other. After that, players have wins, losses, and ties within the round.

For the mathematically inclined, imagine a complete undirected graph, whose nodes are players and edges are battles.

Score calculation

Players win, lose, or tie the round by comparing wins and losses in that round's battles. The table shows the effect on their streak and score.

Wins vs Losses Outcome Effect on Streak Effect on Score
Equal the round is a tie none none
More wins the player wins the round streak becomes 0 if previously negative, and is then incremented gain 1 score point, adjusted by streak bonuses
More losses the player loses the round streak becomes 0 if previously positive, and is then decremented lose 1 score point

Result evaluation

For players to review the round's results after they are displayed, 2 seconds are added to the next round's move selection.

Streak Bonuses

Unless Classic is on, these reward lucky players that win repeatedly, and they also help unlucky players recover from their losses:

  • +5 or higher: see everyone's choices
  • +3 or higher: score multiplier = (streak - 1)
  • -3 or lower: see everyone's choices
  • -5 or lower: if no choice is made, an optimal choice is automatically made

Options

Blitz (quick round)

Move selection happens within 3 seconds instead of 5.

Classic (no streak bonuses)

Streak bonuses are not used in this round, but streaks are still calculated as usual.

Inverted (reverse default damage direction)

The default damage direction is reversed. By this new default, rock beats paper, which beats scissors, which beats rock.

  • Rock beats Paper
  • Paper beats Scissors
  • Scissors beats Rock

Count (damage depends on number of players)

If more people choose a type of move than another one, the loss is negated. If at least twice as many people choose a normally losing move compared to a normally winning move, the damage direction is reversed. Consider these examples:

  • 2 players pick rock, and 2 players picks paper: rock loses to paper, as usual.
  • 3 players pick rock, and 2 players picks paper: rock ties with paper.
  • 4 players pick rock, and 2 players picks paper: rock beats paper.
Players that collude might be able to ruin this gamemode.

Team (maybe in the future)

Players are in two teams. No two players on the same team battle each other; battles apply only to pairs of players on different teams.

For a graph representation, this is a complete bipartite graph.

Survivor (cancelled)

  • Players that just joined recently start with zero health points.
  • Players that just lost a round lose one health point.
  • Players that have zero health points cannot participate.
  • When there is a survivor, all player health points are reset to a positive number.

Cancel (cancelled, reduce common amounts)

This has unknown effect. It would affect score calculation somehow, but I forgot how I wanted to implement it. I think the current rules are as if this is always on.
